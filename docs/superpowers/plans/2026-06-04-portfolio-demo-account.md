# Portfolio Demo Account Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an optional public read-only portfolio demo account with rolling fictional training data generated from safe owner patterns and synthetic capital-city routes.

**Architecture:** The backend owns all security guarantees through an explicit `users.is_demo` flag, a demo-login endpoint, and a reusable read-only guard applied to write routes. Demo data is generated by a deterministic backend service and refreshed by an explicit command plus scheduler support. The frontend adds a polished demo login path and disables high-traffic mutation controls, while backend guards remain authoritative.

**Tech Stack:** FastAPI, SQLAlchemy 2, Alembic, PostgreSQL/SQLite tests, RQ scheduler loop, React, TypeScript, Vite, TanStack Query, Vitest, pytest.
